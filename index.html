Executive Summary

This document outlines the implementation plan for developing a specialized web-based tool focused on collecting, processing, and visualizing charging station data specifically for Electric Heavy-Duty Trucks (E-HDT). The solution addresses critical infrastructure planning needs by providing real-time availability information and analysis of high-capacity charging stations suitable for commercial electric trucking operations.

The proposed system leverages Firebase's real-time database capabilities combined with open-source technologies to deliver a cost-effective yet powerful analysis platform. By focusing specifically on E-HDT requirements, this tool fills a significant gap in current planning resources that typically don't differentiate between passenger vehicle chargers and the high-power infrastructure needed for commercial transport.

E-HDT Charging Specifications & Requirements

Planning Phase

Electric Heavy-Duty Trucks require specialized charging infrastructure with significantly different specifications than passenger EVs. Understanding these requirements is essential for effective data collection and analysis:

Power Requirements

  • Minimum power output of 150kW, with most commercial operations preferring 350kW+ for faster charging
  • Many newer E-HDT models support charging rates up to 1MW with Megawatt Charging System (MCS)
  • Battery capacities ranging from 300-1000kWh, significantly larger than passenger vehicles (50-100kWh)

Connector Types for High-Power E-HDT Charging in Europe

CCS Type 2 Megawatt Charging System (MCS) High Power Charging (HPC)

The European electric heavy-duty truck charging infrastructure relies on three primary high-power connector standards:

  • Combined Charging System (CCS) Type 2
    • Primary charging standard for electric vehicles in Europe
    • Power output range: Up to 500 kW
    • Widely deployed across existing charging networks
  • Megawatt Charging System (MCS)
    • Specifically designed for heavy-duty electric vehicles
    • Power output range: Up to 3.75 MW
    • Emerging standard for ultra-fast charging
    • Developed by a consortium including Daimler, Traton, and Volvo Group
  • High Power Charging (HPC) Connector
    • Advanced charging connector for commercial electric vehicles
    • Power output range: 350 kW to 1 MW
    • Developed by charging infrastructure manufacturers
    • Supports rapid charging for electric trucks and buses

These standards are critical for ensuring interoperability and efficient charging across the European transportation network, particularly along the Trans-European Transport Network (TEN-T) corridors.

Physical Accessibility Requirements

  • Sufficient space for large vehicle maneuvering (minimum 16m turning radius)
  • Pull-through charging layouts rather than parking bay configurations
  • Height clearance accommodations (4.5m minimum)
  • Reinforced pavement to support heavy vehicle weight

Availability Considerations

  • Uptime requirements higher than passenger vehicle network (99%+ target)
  • Multiple charging points per location for redundancy
  • Reservation systems increasingly important for fleet operations

Implementation Note: The Data Team will be responsible for implementing filters based on these specifications to identify charging stations truly suitable for E-HDT operations from the OpenChargeMap API data.

Scope of Implementation with Open Source APIs

Planning Phase

The project will be implemented using entirely open-source technologies and APIs to ensure cost-effectiveness and sustainability:

Core Data Sources

  • OpenChargeMap API provides the foundation for charging station data, offering comprehensive coverage across Europe including the TEN-T network. The API provides station locations, connector types, power levels, and basic availability information.
  • OpenStreetMap API will be used to identify and analyze transport corridors, particularly focusing on routes suitable for heavy-duty vehicles.

Key Open Source Technologies

  • Leaflet.js for interactive mapping without licensing costs
  • GeoFire for optimized geospatial queries within Firebase
  • GeoPandas for spatial data processing and analysis
  • Plotly and Dash for data visualization components
  • Firebase platform (using free tier for development, pay-as-you-go for production)

Implementation Scope Boundaries

  • The system will focus on the TEN-T network corridors within the EU as a priority
  • Real-time availability data will be limited to what OpenChargeMap provides, supplemented by historical analysis
  • The initial version will support Dutch, German, and Italian languages for the UI
  • Mobile responsiveness will be implemented but native mobile apps are outside scope
  • Integration will be primarily through data export rather than direct API connections to third-party systems

This approach ensures the project can be implemented with minimal licensing costs while providing significant value through intelligent processing and visualization of openly available data.

Final Goal

Planning Phase

The ultimate goal of this project is to accelerate the transition to electric heavy-duty transport across European corridors by providing the missing link between raw charging infrastructure data and actionable operational intelligence.

This tool will enable:

  • Data-driven planning of E-HDT operations with confidence in charging availability
  • Identification of infrastructure gaps to guide both route planning and advocacy for new infrastructure
  • Real-time monitoring of critical charging resources to avoid operational disruptions
  • Seamless integration of charging infrastructure data into existing fleet management workflows

By focusing specifically on the unique requirements of E-HDT operations, this solution addresses a critical gap in existing tools that primarily serve passenger vehicle use cases, ultimately helping logistics operators meet sustainability goals while maintaining operational efficiency.

EU-Based Use Case

Planning Phase

As the European Green Deal aims to reduce transport emissions by 90% by 2050, logistics companies operating across the Trans-European Transport Network (TEN-T) corridors face significant challenges in planning E-HDT operations.

Practical Scenario: Transcontinental Logistics Route Planning

A logistics company based in Rotterdam needs to electrify its fleet for routes connecting to Milan via Germany. Fleet managers require:

  1. Accurate information about high-capacity charging stations (350kW+) along the TEN-T corridors
  2. Real-time availability status of stations to schedule charging stops
  3. Identification of infrastructure gaps that might require route adjustments or additional charging stops
  4. Data on physical accessibility for 40-ton trucks including height/space requirements

Currently, the company struggles with fragmented data sources that primarily focus on passenger vehicle infrastructure. The E-HDT Web Scraper tool will allow them to:

  • Identify suitable charging locations along their most frequented routes
  • Monitor real-time availability of these stations
  • Export route-specific data to their existing fleet management system
  • Make data-driven decisions about route planning and charging stops

This use case is particularly relevant as the EU's Alternative Fuels Infrastructure Regulation (AFIR) mandates the development of charging infrastructure specifically for heavy-duty vehicles along the TEN-T network, with requirements for stations every 60km by 2025.

Key EU Regulations
  • European Green Deal: 90% transport emissions reduction by 2050
  • AFIR: HDV charging every 60km on TEN-T network by 2025
  • Euro VII: New emissions standards affecting HDV design
Route Challenges
  • 1,100+ km route length (Rotterdam-Milan)
  • Mountain passes requiring additional energy
  • Cross-border operations through 3+ countries
  • Different charger network operators in each region

Detailed Deliverables

Planning Phase

The project will produce the following specific deliverables:

1. Data Collection Module

  • Python-based API connector for OpenChargeMap with authentication handling
  • E-HDT-specific filtering system based on:
    • Power output thresholds (≥150kW, ≥350kW, ≥1MW categories)
    • Compatible connector types (CCS2, MCS)
    • Physical accessibility parameters
  • Real-time availability monitoring system with configurable polling intervals
  • Data validation and cleaning pipeline for ensuring quality

2. Firebase Integration

  • Custom Firebase Realtime Database schema optimized for geospatial queries
  • GeoFire implementation for efficient location-based data retrieval
  • Cloud Function-based automation for scheduled data collection and processing
  • Authentication system with role-based access controls

3. Analysis Engine

  • Corridor identification system using OpenStreetMap data
  • Coverage gap detection along major transport routes
  • Charging time calculation based on E-HDT battery specifications
  • Optimal charger placement recommendation algorithm
  • Time-based availability pattern analysis

4. Visualization Dashboard

  • Interactive web interface with responsive design
  • Leaflet.js-based map with custom layers for:
    • Charger locations with real-time status indicators
    • Coverage analysis along corridors
    • Charging infrastructure density visualization
  • Advanced filtering system for charger properties
  • Customizable dashboard with savable configurations
  • User account management system

5. Export System

  • Format-specific export handlers for:
    • CSV for spreadsheet analysis with custom field selection
    • KML for Google Earth visualization with style configurations
    • Shapefile (SHP) for GIS software with attribute mapping
  • Custom report generator with visualization snapshots
  • Integration hooks for external fleet management systems

6. Documentation and Training Materials

  • Comprehensive system documentation including API references
  • User guides with scenario-based examples
  • Administrator documentation for system maintenance
  • Video tutorials for core functionality
